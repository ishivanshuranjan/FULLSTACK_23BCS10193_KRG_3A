User Roles:

Admin: create/manage sessions, view attendance logs, export data (CSV/PDF)

User: scan QR code to mark attendance (check-in/check-out)

QR Code Handling:

Backend generates QR codes using ZXing for each session.

QR payload format:
SESSION_<UUID>;TYPE=<checkin|checkout>;EXP=<timestamp>

Each QR has a short expiry (e.g., 5 mins).

Frontend displays this QR on the admin dashboard.

Frontend users scan using react-qr-reader.

Attendance Flow:

On scan, frontend extracts payload and sends it with JWT token to backend:

POST /api/attendance/check
{
  "sessionId": "<uuid>",
  "type": "checkin" or "checkout",
  "qrData": "<payload>"
}


Backend validates:

Token authenticity (JWT)

Session validity (active + not expired)

Prevent duplicate check-ins/outs

If valid:

Log entry in attendance_logs:

{
  userId,
  sessionId,
  checkInTime / checkOutTime,
  userAgent,
  ipAddress
}
